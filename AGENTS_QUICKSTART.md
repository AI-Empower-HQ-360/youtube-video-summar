## ğŸ¯ Best Practices

### Before Creating a PR:

```bash
# 1. Run tests locally
npm test

# 2. Check linting
npm run lint

# 3. Check types
npm run type-check

# 4. Build successfully
npm run build
```

### Responding to Agent Feedback:

1. **Review the comments** - Agents post findings on your PR
2. **Fix issues** - Address errors and warnings
3. **Re-push** - Agents re-run automatically
4. **Repeat** until all checks pass

### Ignoring False Positives:

If an agent flags something incorrectly:

```typescript
// eslint-disable-next-line rule-name
const myCode = something;
```

Or add to `.eslintignore`, `.gitignore`, etc.

## âš™ï¸ Configuring Agents

### Code Review Rules

Edit `.github/agents/code-review.config.json`:

```json
{
  "rules": {
    "complexity": {
      "maxCyclomaticComplexity": 10,  // Adjust complexity limit
      "maxFileLines": 300              // Adjust file size limit
    }
  }
}
```

### Test Coverage Thresholds

Edit `.github/agents/testing.config.json`:

```json
{
  "coverage": {
    "global": {
      "statements": 80,  // Change minimum coverage %
      "branches": 75,
      "functions": 80,
      "lines": 80
    }
  }
}
```

### Security Policies

Edit `.github/agents/security.config.json`:

```json
{
  "scanners": {
    "npm-audit": {
      "severity": ["critical", "high"],  // Adjust severity levels
      "autoFix": true                     // Enable/disable auto-fix
    }
  }
}
```

## ğŸ› Troubleshooting

### Agent Not Running?

1. **Check workflow file** - Is it enabled?
2. **Check triggers** - Does your branch match?
3. **Check permissions** - Are Actions enabled?
4. **Check logs** - Any errors in workflow?

### Agent Failing?

1. **Check logs** in GitHub Actions
2. **Review configuration** files
3. **Test locally** using same commands
4. **Check dependencies** are installed

### Too Many Alerts?

1. **Adjust thresholds** in config files
2. **Add ignore rules** for false positives
3. **Improve code quality** to reduce warnings
